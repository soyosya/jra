5.3 履歴の抽出順序
1. 今走馬ごとに、開催日 < 対象日の過去走を取得する。2. 着順 > 0、帯広ばを除外する。3. 開催日 DESC、レース番号 DESCで並べる。4. ROW_NUMBER()で1～3位の直近3走だけを残す。5. rn=1を直前走としてsame、S1、時計、前走ADV/HOLDに使う。

5.4 馬の同定
現行SQLは馬名でJOINし、双方に血統登録番号がある場合は一致を要求する。血統登録番号が片側欠損の場合は馬名一致だけで許可される。これは実装上の正本であるが、同名馬の誤結合リスクが残るため、運用ログには血統登録番号の有無を記録することが望ましい。

6. 前処理と相対化
6.1 各コーナー相対位置
pr1 = (前走1角順位 - 1) / (前走頭数 - 1)pr2 = (前走2角順位 - 1) / (前走頭数 - 1)pr3 = (前走3角順位 - 1) / (前走頭数 - 1)pr4 = (前走4角順位 - 1) / (前走頭数 - 1)有効条件: 前走頭数 > 1 かつ 対象コーナー順位 > 0

6.2 直近3走平均
a3_3とa4_3は、直近3走のうち取得できた値だけを個別に平均する。現行本番コードでは、3角と4角を必ず同じ過去走のペアにそろえる処理はしていない。つまりa3_3とa4_3で有効走数が異なる場合があり得る。この挙動を黙って修正してはならない。
a3_3 = mean(直近3走の有効なpr3)a4_3 = mean(直近3走の有効なpr4)有効値が1つ以上あれば平均を作る。

既知仕様
ペア走だけで平均する頑健版は研究候補であり、現行S判定の正本ではない。変更する場合は新バージョンとして比較する。

6.3 今走馬番相対
waku_relative = (今走馬番 - 1) / (今走頭数 - 1)通過条件: waku_relative < 0.800.80ちょうどは不通過。<= ではない。

7. レース適格性ゲート
馬単位の判定に入る前に、レース単位で次のゲートを順番どおりに通す。順序を変えても最終集合が同じ場合はあるが、ログと不発理由を一意にするため処理順序を固定する。

8.2 A3-RELの意味
A3-RELは「この馬の絶対的な3角位置」を中団と判定するものではない。同条件組の今回メンバーの中で、過去3走平均3角位置が中位1/3にある馬を抽出する相対定義である。少頭数やメンバーの脚質構成による見かけの差を、同条件組内順位で処理する。

9. S1持続とM_base
9.1 S1持続
M_rawの各馬について、直前走の1角・2角・3角相対位置がすべて中団帯に残っていたかを確認する。
S1_pass =    0.30 <= p1 <= 0.70AND 0.30 <= p2 <= 0.70AND 0.30 <= p3 <= 0.70境界0.30と0.70は通過する。

p1、p2、p3のどれか一つでも欠損ならS1不通過である。平均値や補完値で代用しない。
9.2 今走馬番相対フィルター
waku_relative < 0.80

同条件組の中団再現検証で採用されている現行条件である。観察起点であり将来のOOS監視対象だが、前向き検証中は変更しない。
9.3 M_baseの正式定義
M_base = { h in M_raw | S1_pass(h) AND waku_relative(h) < 0.80 }本番コード上の名称: mids

9.4 繰り上げ禁止
M_raw上位の馬がS1や枠条件で落ちても、別の位置帯の馬を中団へ繰り上げない。M_rawはA3-RELで固定し、その中から資格を落とすだけである。

10. 同条件組・前走時計1位による能力選抜
能力選抜の正本は、汎用加重スコアではなく「sameH内で直前走走破時計が最小の馬」である。sameHは全出走馬ではなく、直前走が今走と同一競馬場・同一距離だった馬だけで構成される。
10.1 clock1の計算
withT = sameHのうち、前走時計が存在し、前走時計 > 0 の馬withTを 前走時計 ASC, 馬番 ASC で並べるt1u = 先頭馬の馬番

10.2 同時計の現行処理
前走時計が同一の場合、現行本番コードは馬番が小さい馬を時計1位にする。時計同率全頭を1位扱いにはしない。これは能力差を表す処理ではなく、決定性を保つための実装上のタイブレークである。
既知の限界
clock_tieを別列で記録し、unique_clock1との前向き比較を推奨する。ただし本番条件を途中で変えず、旧定義を維持する。

10.3 有力馬候補の発火条件
if t1u is not in M_base:    pick_u = blank    tier = blank    別のM_base馬を繰り上げないif t1u is in M_base:    pick_u = t1u    tier = A から開始し、S条件を評価する

10.4 能力順位を別指数で置換しない
コンピ指数1位、上がり1位、独自能力点などをclock1の代替として自動使用しない。これらは別ロジックまたはタイブレーク研究であり、正式V3縮約版の能力中核はclock1である。

平均的にも直前走でも、3角から4角に相対位置を前進させていることを要求する。位置維持（差=0）は不通過である。

final_rel = (前走着順 - 1) / (前走頭数 - 1)simple_HOLD_pass = final_rel < p4

直前走の4角から最終着順まで、さらに相対位置を前進させた場合だけ通過する。4角順位をそのまま維持した場合（final_rel = p4）は現行簡易HOLDでは不通過である。

S_pass = simple_ADV_pass AND simple_HOLD_passtier = "S" if S_pass else "A"

誤解防止
第三者評価で検討した「HOLD >= 0」や符号再現2/3走は頑健版の研究定義である。現行本番Sは「前走4角から着順まで厳密に前進」で固定する。

19. 買目生成手順
以下は「中団確度選抜を既存買目に統合する正準手順」である。既存ロジックの◎○▲△や券種テンプレートを入力として扱い、未定義の閾値を本書で新設しない。
19.1 正準手順
既存予想エンジンを先に実行し、主軸、相手、券種候補、見送り、信頼区分を確定する。
同じ発走前データ時点で本中団ロジックを実行し、M_base、pick_u、tierを付与する。
pick_uと既存主軸の一致／不一致を判定する。
第18章の決定表に従い、信頼度注記またはレビュー候補を付ける。
既存の点数上限・二軸条件・相手条件を通過した場合だけ買目候補へ反映する。
発走前オッズを取得し、既存EVゲートを適用する。オッズがなければ購入ではなく候補表示に止める。
購入／見送り理由と、S/Aが買目に与えた変更をログへ保存する。
19.2 候補表示モード（オッズなし）
オッズがない場合は「購入推奨」と表示しない。券種と組合せは参考候補として表示できるが、金額と購入可否は未判定とする。
19.3 参考表示テンプレート
この会話で全レースを一覧表示した際の簡易テンプレートは、既存の正式買目仕様が提供されていない場合の参考表示であり、公式購入ロジックではない。

付録E. 最終要約
正式ロジック
対象20セルで、同条件率60％以上・nm5以上のレースからA3-REL中位1/3を作り、前走1～3角が0.30～0.70かつ今走馬番相対<0.80のM_baseを確定する。sameHの前走時計1位がM_baseならA、さらに平均3→4角前進・前走3→4角前進・前走4角→着順前進をすべて満たせばS。

買目
S/Aは確度タグであり、既存主軸と一致した場合の信頼補強を主用途とする。別馬なら自動置換せず、二軸または相手候補のレビューへ回す。オッズ／EVゲートを通過しない限り実購入しない。

禁止
中団先頭、先行圧力、中団密度、任意加重、欠損補完、繰り上げ、Sだけの全件購入を正式ロジックへ戻さない。
