# 高知競馬 分析ロジック ガイド

高知競馬(NAR)の予想・回収率分析の知見とスクリプト群をまとめたもの。
**別セッション/別メンバーは、まずこのファイルを読めば全体を再現・継続できる。**

対象データ: 2022-01 以降が充実(直近まで)。分析は原則 **2024-01 以降**を使用。

---

## 0. 環境・接続

- **DB接続**(PowerShell `System.Data.SqlClient`): 接続文字列は `共通/appsettings.json` の
  `ConnectionStrings.DefaultConnection` から読み込む(SQL Server `192.168.168.81\SQLEXPRESS` / DB `中央競馬`)。
  資格情報はリポジトリに直書きせず appsettings 側で管理すること。
- **実行**: Windows PowerShell 5.1。各スクリプトは `tools/` に置く。
- **主要テーブル**:
  - `競走結果`: 着順・走破時計(秒)・上り3F・一着馬着差タイム・先着/後着馬着差タイム・一〜四コーナー通過順。
  - `レース情報`(出馬表由来): 騎手・調教師・馬主・距離・条件・一着賞金(=クラス)・馬名・馬番・枠番・馬体重・馬体重増減・斤量・斤量増減・馬場・天候。
  - `払戻金`: 馬券・組番・金額(¥100あたり払戻)。
  - その他: 当日メニュー・開催情報・馬情報(血統)・リアルタイムオッズ。

### 環境の落とし穴(必読)
- **日本語を含む .ps1 は BOM付きUTF-8で保存**しないと PS5.1 で文字化け。保存後に次で付与:
  ```powershell
  $p="...ps1"; $t=[IO.File]::ReadAllText($p,[Text.UTF8Encoding]::new($false)); [IO.File]::WriteAllText($p,$t,[Text.UTF8Encoding]::new($true))
  ```
  (BOMの二重混入=U+FEFF重複に注意)
- **PS5.1の罠**: `@($list) | Where-Object {...}` を関数引数に直接渡すと "Argument types do not match" 例外。**`$arr.Where({...})` メソッド**を使う。`Add-Member` は **`-NotePropertyName`/`-NotePropertyValue` を明示**(位置引数は MemberType 解釈で失敗)。インライン `if` 式は `$( if(...){}else{} )`。
- **SQLのテーブル別名 `ri` は禁止**(Remove-Item と誤検知されブロック)→ `rinfo` 等にする。
- **`レース情報.着順` は常に0で未使用**。着順は `競走結果` と `開催場所+開催日+レース番号+馬番` で結合して取得。
- **破損タイム**(例: 1400mで60秒台)が混入 → 距離別の妥当速度域 `距離/18 〜 距離/11 [m/s]` でフィルタ。
- **オッズ/人気はDB未保存**(`リアルタイムオッズ` は高知0行)→ 回収率は `払戻金` で近似(脚質別の機械買い・本命1点の払戻ベース)。
- **血統**(`馬情報` の父/母父)は高知出走馬の **約13%しか無く偏在** → 分析不可。

## 1. 戦略的な結論(最重要)

1. **前残りバイアスは巨大**: 今走逃げ 勝率33%/単勝回収205%、先行16%/93%、差し9%、追込3%。短距離・重不良馬場ほど極端(不良で逃げ勝率38%・追込ほぼ全滅)。
2. **しかし高知市場は"馬の強さ"に効率的**: 前走脚質・能力(補正タイム)・体調・クラスを機械買いしても**回収率≒控除率(73〜75%)**。馬の地力はオッズに正確に織り込まれている。
3. **妙味は"展開の歪み"にある**: 市場が読みにくい「前残り」を当てると+EV。具体的には
   - **同型不在**(他に前走前々の馬が少ない)→ 単独逃げ濃厚なのに人気が割れる。
   - **隠れ前残り**(前走差し/追込なのに、調教師×馬主×騎手のコネクションが前傾)→ 穴の逃げ。
4. **馬場差補正**: 日次δ(同日全レースの時計水準の中央偏差、レンジ約7.8秒)で補正すると、同条件の勝ち時計SDが44%縮小。能力評価の土台。
5. **較正は良好**: 馬軸スコアの推定勝率%は実勝率とほぼ一致(信頼できる数値)。
6. **回収率100%超はまだ点推定**: 単勝は分散が大きく(数本の万馬券に依存)、95%CIは100%をまたぐ。**実戦トラッキング継続が前提**。本命はスコア上位で絞る運用(閾値を上げるほど回収率↑)。

## 2. スクリプト一覧(高知)

すべて `tools/` 配下。引数・知見・実行例を併記。`-From` 既定は 2024-01-01。

| スクリプト | 目的 | 主な引数 |
|---|---|---|
| `kochi-score.ps1` | 指定日の総合スコアリング(脚質×継続×騎手前付×持時計) | `-Date yyyy-MM-dd` |
| `kochi-roi.ps1` | 脚質別に単勝/複勝を機械買いした払戻ベース回収率 | `-From -To` |
| `kochi-track-variant.ps1` | 日次馬場差δの算出と検証(補正前後SD) | `-From -RecentDays` |
| `kochi-lead-transition.ps1` | 「今走逃げ」の規定要因(前走脚質・乗替・枠・上り3F) | `-From` |
| `kochi-lead-connections.ps1` | 調教師×馬主×騎手(三者コネ)×前走 の逃げ深掘り | `-From -MinN -Thresh` |
| `kochi-backtest.ps1` | 改良スコア(同型不在+補正持時計+価値コネ+馬場)の本命バックテスト | `-TrainEnd -TestStart -Refresh` |
| `kochi-horse-score.ps1` | 馬軸の較正勝率%(A:馬ファクトのみ / B:展開統合) | `-TrainEnd -TestStart -Refresh -ShowRace yyyy-MM-dd` |

### 各スクリプトの要点

- **kochi-roi.ps1** — 前残りが「金額」で儲かるか検証。結論: 今走逃げ単勝205%だが前走脚質の機械買いは74.7%(織込済)。基準=全頭買い 単勝73.7%/複勝69.0%。
- **kochi-track-variant.ps1** — 期待タイム=距離×クラスの平均勝ち時計、偏差の日次中央値=δ。補正後タイム=生タイム−δ。検証で勝ち時計SDが1.62→0.91s(44%↓)。
- **kochi-lead-transition.ps1** — 前走脚質が最大予測子。乗替は**前付け騎手(前々率40%↑)への乗替時のみ**逃げ率倍増(6.5→15.4%)。枠順・内外シフトは無関係。
- **kochi-lead-connections.ps1** — 「強コネ」(三者の延べ≥30走 & リーブワンアウト逃げ率≥20%)は逃げ率30.8%(非強コネ9.3%)。前傾厩舎=打越勇/東原己/國澤輝、前傾馬主=中村智幸/ウエスト.フォレスト。強コネ×前走逃げ55%、後ろ脚質の強コネが逃げると激アツ(差し起点392%)。
- **kochi-backtest.ps1** — 検証2025-10〜(約900R)本命1点: 現行相当91% → 改良(同型不在+距離)101% → **改良+価値コネ+馬場 108.6%**。閾値≥4で121.9%。**注意**: コネを素朴加点すると的中↑回収↓(強コネ=人気で織込済)。「隠れ前残り(前走差/追なのに強コネ)」にのみ加点するのが正解。特徴量は `_kochi_feat_cache.csv` にキャッシュ(`-Refresh` で再抽出)。
- **kochi-horse-score.ps1** — 各馬に **較正済み勝率%** を付与。A(能力/体調/クラス/相性のみ)=73.6%(織込済)、B(+展開)=**116.1%**。較正良好(予測%≒実勝率)。`-ShowRace` で指定日の各馬スコア/勝率%一覧。キャッシュ `_kochi_horse_cache.csv`。馬体重増減の符号はDB上未確認(現状|Δ|のみ使用)。

### 共通の特徴量定義
- **脚質**: `early_pos = COALESCE(NULLIF(一コーナー,0),NULLIF(二,0),NULLIF(三,0),NULLIF(四,0))`。逃げ=1、先行≤頭数×0.33、差し≤0.66、追込=それ以降。`頭数`=そのレースの着順>0の数。
- **騎手前々率(前付力)**: (逃げ+先行)率。リーク回避のため学習期間のみで算出。
- **同型頭数(mae)**: 同レースの「前走前々(逃げ/先行)」馬の数。少ないほど前残り濃厚。
- **補正後持時計(能力)**: 同距離・365日内の自己ベスト `MIN(走破時計 − δ)` を 距離×クラス基準と比較。

## 3. 標準的な使い方

```powershell
# 1) 過去検証(モデルの回収率を確認。-Refresh は初回/特徴量変更時のみ)
.\tools\kochi-backtest.ps1 -Refresh
.\tools\kochi-horse-score.ps1 -Refresh

# 2) 指定日の各馬の勝率%を見る
.\tools\kochi-horse-score.ps1 -ShowRace 2026-06-26

# 3) 当日の総合スコア
.\tools\kochi-score.ps1 -Date 2026-06-26
```

実戦運用の指針: **B(展開統合)の勝率%で序列付け**し、本命はスコア上位で絞る。妙味は「能力評価(A)が高いのに人気が割れる/前走が地味な前残り候補」を狙う。回収率100%超は点推定なので少額で継続トラッキング。
